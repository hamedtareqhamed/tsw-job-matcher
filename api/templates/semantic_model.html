Semantic Web Model & Architecture

Technical documentation of the RDF schema, OWL axioms, and SPARQL queries underpinning the IT Job Matcher.

Semantic Web Technology Stack

RDF Triples & URIs

Data is stored as a directed graph of Subject-Predicate-Object triples. Every resource is represented by a unique URI, establishing an unambiguous web of connected entities.

OWL Ontology Schema

Defines classes, subclasses, properties, and axioms (such as disjoint classes, functional property bounds, and inverse property mappings).

SPARQL Graph Querying

A declarative query language used by the python middleware to bind variables and match triple patterns against the loaded ontology.

OWA vs CWA Logical Logic

Reconciles Open-World Assumption (Semantic layer) with Closed-World Assumption (Python execution layer) to detect missing skills under Negation-as-Failure.

XSD Datatype Safety

Binds literal values (such as graduation years and experience durations) to strict XML Schema Datatypes like xsd:integer and xsd:string.

Ontology Vocabulary & Structure

The ontology defines six primary classes, specific skill category subclasses, and custom properties governing the domain of IT job matching:

Core Classes & Taxonomy:
  • ex:Applicant — Candidate profiles.
  • ex:Job — Employment postings.
  • ex:Company — Hiring organisations.
  • ex:Education — Academic degrees.
  • ex:Experience — Professional internships/roles.
  • ex:Skill — Base technical skill class.
  • ex:ProgrammingLanguage — Subclass of ex:Skill.
  • ex:AIDataSkill — Subclass of ex:Skill.
  • ex:WebCloudSkill — Subclass of ex:Skill.
  • ex:SecuritySkill — Subclass of ex:Skill.
  • ex:DevToolSkill — Subclass of ex:Skill.
Key Properties (Axioms):
  • ex:hasSkill (Domain: ex:Applicant, Range: ex:Skill)
  • ex:requiresSkill (Domain: ex:Job, Range: ex:Skill)
  • ex:requiresCoreSkill (Domain: ex:Job, Range: ex:Skill, Sub-property of: ex:requiresSkill)
  • ex:offeredBy (Domain: ex:Job, Range: ex:Company, Type: owl:FunctionalProperty)
  • ex:offersJob (Domain: ex:Company, Range: ex:Job, Inverse of: ex:offeredBy)

Active SPARQL Query Examples

Below are two real SPARQL queries executing inside our Flask app backend using RDFLib.

Query 1: Retrieving the Skills Taxonomy dynamically (Lecture 06 Subclasses)

This query navigates the RDFS class hierarchy to discover all subclasses of ex:Skill, and then retrieves all skill instances belonging to those classes. 

PREFIX ex: <http://example.org/jobmatch#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT ?skill ?label ?catLabel
WHERE {
    ?cat rdfs:subClassOf ex:Skill .
    ?skill rdf:type ?cat .
    ?skill rdfs:label ?label .
    ?cat rdfs:label ?catLabel .
}
ORDER BY ?catLabel ?label
Academic Connection: Implements the rdfs:subClassOf reasoning described in Lecture 06 (RDFS). It allows the web application to automatically catalog any new skill classes or instances added to the Turtle file.
Query 2: Fetching Core and Secondary Required Skills (Lecture 08 UNION & BIND)

Runs a graph pattern matching union (UNION) with variable bindings (BIND) to identify which skills are classified as Core vs Bonus/Secondary. 

PREFIX ex: <http://example.org/jobmatch#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?skill ?label ?isCore
WHERE {
    {
        ?job ex:requiresSkill ?skill .
        BIND(false AS ?isCore)
    } UNION {
        ?job ex:requiresCoreSkill ?skill .
        BIND(true AS ?isCore)
    }
    ?skill rdfs:label ?label .
}
Academic Connection: Demonstrates graph pattern alternative matchers (UNION) and variable assignment constructors (BIND) from Lecture 08 (SPARQL Part 2). It maps ontological relations to weights dynamically.
