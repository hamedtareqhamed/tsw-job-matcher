Ontology-Based Match Rules & Logical Constraints
Skill Weighting (Core vs Secondary):

Core domain skills carry 70% of the score. Secondary or tooling skills (e.g. Git) carry 30%. A matching status cannot be "Good/Excellent Match" unless you satisfy at least one core skill.

Semantic Skill Subsumption (Inference Rules):

The system automatically infers prerequisite skills. If you possess React, it infers JavaScript, HTML, CSS. If you possess Deep Learning, it infers Machine Learning & Python.

Experience Level Constraints:

Junior level jobs require at least 12 months of experience. Falling short applies a -10% experience penalty, while satisfying it applies a +10% match bonus.

Location Proximity Rules:

Proximity-based matching within Klang Valley (KL, PJ, Cyberjaya) allows commute with a tiny -5% modifier. Relocation to other regions (e.g. Penang) applies a -20% penalty.
